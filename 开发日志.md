# 项目开发进展 - OopsOS（对不队）

---

## **项目起步** - 2025年11月13日

### **项目目标：**
- 基于 `xv6` 操作系统的架构，开发一个简化、模块化的操作系统，命名为 **OopsOS**。
- **功能目标：** 提供基本的操作系统功能，如进程管理、内存管理、文件系统和设备驱动等。
- **学习目标：** 通过开发，深入理解操作系统的原理和体系结构。

### **技术选型：**
- **基础框架：** 基于 `xv6` 操作系统，便于扩展与学习。
- **架构支持：** RISC-V 架构，增强系统可移植性和现代硬件支持。
- **编程语言：** C语言（与 `xv6` 保持一致）。

---

## **第一阶段：基础系统架构搭建** - 2025年11月15日

### **开发环境搭建：**
- 安装并配置了 RISC-V 工具链和 QEMU 模拟器，确保能够构建并运行 `xv6`。
- 成功克隆了 `xv6` 源代码，完成编译与测试。
- 配置了本地开发环境，包括使用 IDE（如 VS Code 或 Vim）及调试工具。

### **系统架构设计：**
- 在 `xv6` 基础上设计了初步架构，确保能够运行于 RISC-V 架构，并支持模块扩展。

---

## **第二阶段：模块开发与功能完善** - 2024年11月17日

### **进展与功能：**

- **11/17:**  
  - 实现系统调用 `SYS_cps` ：可查看当前系统的进程信息（包括进程名、PID、优先级等）。

- **11/20:**  
  - 实现系统调用 `SYS_trace` ：能够指定并跟踪系统调用。

- **11/21:**  
  - 实现系统调用 `SYS_sysinfo` ：可查看当前进程数量及剩余空闲内存。
  - **内存管理：** 实现写时分配技术（Copy-On-Write）。

- **11/24:**  
  - **锁机制优化：** 将空闲页面链表和 Buffer Cache 的锁进一步细化，减小锁颗粒度，提高并行效率。

- **11/25:**  
  - **内存管理：** 修改系统调用 `SYS_bark`，并实现懒分配功能。
  - 改写相关用户测试程序。

- **11/26:**  
  - **进程调度：** 修改为动态优先级调度算法：
    - 实现系统调用 `SYS_setPriority` ：支持动态调整进程优先级。
    - 修改进程结构，使其在每次中断时动态调整运行信息。
    - 提高了响应性与公平性，更适应负载变化。

- **11/27:**  
  - 实现系统调用 `SYS_execve` ：执行指定路径的程序。

- **11/31:**  
  - 优化调度算法，修复创建大量子进程可能导致死机的 BUG，提高系统健壮性。

- **12/1:**  
  - 实现系统调用 `SYS_getparentpid` ：获取当前进程的父进程 PID。
  - 实现系统调用 `SYS_print_pgtable` ：打印当前进程的页表信息。

- **12/3:**  
  - **文件管理 & 内存管理：** 实现内存映射文件功能：
    - 支持将文件直接映射到用户空间，比标准 I/O 更高效。
    - 实现系统调用 `SYS_mmap`  和 `SYS_munmap` ，用于建立/取消文件映射。

- **12/4:**  
  - **信号量功能开发：** 实现信号量机制，添加以下系统调用：
    - `SYS_sem_create`: 创建信号量，初始化资源计数。
    - `SYS_sem_free`: 释放信号量，清除资源占用。
    - `SYS_sem_p`: 执行信号量的 P 操作，减少资源计数。
    - `SYS_sem_v`: 执行信号量的 V 操作，增加资源计数。
    - `SYS_sh_var_read`: 读取共享变量的值。
    - `SYS_sh_var_write`: 写入共享变量的值。
  - 完成相关测试代码的编写与调试，修复多次 `fork` 引发的 `sched locks` 问题。
- **12/11:**  
  - **文件管理：** 完善混合索引分配，在原有基础上添加二级间接块
  - 实现符号链接，当一个符号链接打开时，内核跟随该链接指向引用的文件。
    - `SYS_symlink`: 创建符号链接
    - `SYS_create` : 新建一个文件
- **12/12:**
  - **共享内存功能开发：**
    - 实现了共享内存的创建、映射、访问和释放功能，支持进程间共享内存。
    - 添加了以下系统调用：
      - `SYS_shmgetat`: 获取共享内存并将其映射到进程空间。
      - `SYS_shmrefcount`: 查询共享内存的引用计数。
    - 实现了共享内存的引用计数管理、内存分配、映射、解除映射等功能，确保进程间共享内存的同步与资源回收。
    - 完成了父子进程使用共享内存的示例，展示了父进程和子进程如何访问和修改共享内存中的数据。
- **12/17:**
  - **消息队列功能开发：**
    - 实现了消息队列的创建、发送、接收功能,支持进程间通信。
    - 实现了消息队列的创建、发送消息到队列、从队列接收消息的基本功能。
    - 支持消息队列的内存管理，避免了内存碎片。
    - 实现了进程在队列满时的写阻塞和在队列空时的读阻塞机制
    - 添加了以下系统调用：
      - `SYS_mqget`: 获取消息队列。
      - `SYS_msgsnd`: 发送消息到消息队列。
      - `SYS_msgrcv`: 从消息队列接收消息。
- **12/18:**
  - **中断：** 实现`SYS_sigalarm` 和`SYS_sigreturn` 两个系统调用，为用户进程添加定期通知功能，使得进程在一段时间内使用 CPU 后，会被定期“提醒”类似于一种用户态的中断处理，用来模拟用户级的异常处理。
- **12/19:**
  - **网络模块：** 添加e1000网卡驱动，实现对UDP/IP协议的支持
  - 实现系统调用`SYS_connect` ：通过给定的远程地址、本地端口和远程端口来创建一个套接字连接。
- **12/19:**
  - **文件系统：** 实现文件访问控制权限：读、写、执行
  - 实现系统调用`SYS_chmod` ：改变文件的访问权限
- **12/21:**
  - **文件系统：** 实现恢复已删除文件的功能
      - 添加了以下系统调用：
        - `SYS_geti` : 保存文件的索引信息。
              - `SYS_recoveri` : 根据文件的索引信息恢复文件。
- **12/30:**
  - **进程同步/信号量完善：**
    - 允许信号量初值为 0，支持阻塞语义
    - 调整 P/V 休眠与唤醒锁使用，避免丢失唤醒与羊群效应
    - 释放时检查等待者，避免误释放导致进程挂起
    - 修复 `wakeup`/`wakeupOneProc` 锁释放问题（panic: release/acquire）
    - 新增 `semtest` 用例并接入 UPROGS
    - 扩展 `semtest` 覆盖非法参数、释放后访问、多等待者与并发计数场景
    - 将 `semtest` 纳入 `usertests` 自动测试集
    - 新增 AND 信号量接口 `sem_p_multi`，支持原子申请多个信号量
    - 增加 `semandtest` 测试与 `usertests` 集成，展示有/无 AND 的差异
