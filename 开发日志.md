# 项目开发进展 - OopsOS
---

## **项目起步** - 2025年1月3日
### **项目目标**
- 基于 `xv6` 操作系统的架构，开发一个简化、模块化的操作系统，命名为 **OopsOS**。
- **功能目标：** 提供基本的操作系统功能，如进程管理、内存管理、文件系统和设备驱动等。
- **学习目标：** 通过开发，深入理解操作系统的原理和体系结构。
### **设计特点**
- **基础框架：** 基于 `xv6` 操作系统，便于扩展与学习。
- **架构选择：** RISC-V 架构，增强系统可移植性和现代性。
- **编程语言：** C 语言（与 `xv6` 保持一致）。
---

## **第一阶段：基础系统架构搭建** - 2025年1月5日
### **开发环境搭建：**
- 安装并配置 RISC-V 工具链和 QEMU 模拟器，能构建并运行 `xv6`。
- 成功克隆 `xv6` 源代码，完成编译与测试。
- 配置本地开发环境，包括使用 IDE（如 VS Code/Vim）及调试工具。
### **系统架构设计：**
- 在 `xv6` 基础上设计初始架构，确保能够运行于 RISC-V 架构，并便于模块扩展。
---

## **第二阶段：模块开发与功能完善** - 2024年1月7日
### **进展与功能：**

- **11/17:**
  - 实现系统调用 `SYS_cps`：查看当前系统进程信息（进程名、PID、优先级等）。
- **11/20:**
  - 实现系统调用 `SYS_trace`：指定并跟踪系统调用。
- **11/21:**
  - 实现系统调用 `SYS_sysinfo`：查看当前进程数量与剩余空闲内存。
  - **内存管理：** 实现写时复制（Copy-On-Write）。
- **11/24:**
  - **锁机制优化：** 细化空闲页面链表和 Buffer Cache 的锁，降低锁粒度，提高并行效率。
- **11/25:**
  - **内存管理：** 增加系统调用 `SYS_sbrk`，实现懒分配。
  - 改写相关用户测试程序。
- **11/26:**
  - **进程调度：** 改为动态优先级调度算法。
    - 实现系统调用 `SYS_setPriority`，支持动态调整优先级。
    - 扩展进程结构，支持运行信息动态调整。
    - 提升响应性与公平性，更适应负载变化。
- **11/27:**
  - 实现系统调用 `SYS_execve`：执行指定路径程序。
- **11/31:**
  - 优化调度算法，修复大量子进程创建导致死机的 BUG，提高系统健壮性。
- **12/1:**
  - 实现系统调用 `SYS_getparentpid`：获取父进程 PID。
  - 实现系统调用 `SYS_print_pgtable`：打印当前进程页表信息。
- **12/3:**
  - **文件管理 & 内存管理：** 实现文件内存映射。
    - 支持将文件映射到用户空间，比标准 I/O 更高效。
    - 实现系统调用 `SYS_mmap` / `SYS_munmap` 用于建立与取消映射。
- **12/4:**
  - **信号量功能开发：** 完成信号量机制。
    - `SYS_sem_create`、`SYS_sem_free`、`SYS_sem_p`、`SYS_sem_v`
    - `SYS_sh_var_read`、`SYS_sh_var_write`
  - 编写并调试相关测试代码，修复 `fork` 引发的 `sched locks` 报错。
- **12/11:**
  - **文件管理：** 完善混合索引分配，新增二级间接块。
  - 实现符号链接：
    - `SYS_symlink`：创建符号链接
    - `SYS_create`：新建文件
- **12/12:**
  - **共享内存功能开发：**
    - 支持共享内存的创建、映射与释放。
    - `SYS_shmgetat`、`SYS_shmrefcount`
    - 实现引用计数与回收，支持父子进程共享示例。
- **12/17:**
  - **消息队列功能开发：**
    - 实现创建、发送、接收与阻塞机制。
    - `SYS_mqget`、`SYS_msgsnd`、`SYS_msgrcv`
    - 优化内存管理，避免碎片。
- **12/18:**
  - **中断/信号：** 实现 `SYS_sigalarm` / `SYS_sigreturn`，支持用户进程定期通知与用户态异常处理模拟。
- **12/19:**
  - **网络模块：** 添加 e1000 网卡驱动，支持 UDP/IP 协议。
    - `SYS_connect`：基于远程地址/端口建立连接。
- **12/19:**
  - **文件系统：** 实现文件访问权限（读/写/执行）。
    - `SYS_chmod`：修改访问权限。
- **12/21:**
  - **文件系统：** 实现已删除文件恢复。
    - `SYS_geti`：保存文件索引信息
    - `SYS_recoveri`：根据索引信息恢复文件
- **12/30:**
  - **进程同步/信号量完善：**
    - 允许初值为 0，支持阻塞等待。
    - 调整 P/V 休眠与唤醒锁，避免丢失唤醒和羊群效应。
    - 释放时检查等待队列，防止重复释放导致挂死。
    - 修复 `wakeup`/`wakeupOneProc` 锁释放问题（`panic: release/acquire`）。
    - 新增 `semtest` 并接入 `UPROGS` / `usertests`。
    - 扩展 `semtest`，覆盖非法参数、释放后访问、多等待者与并发计数。
    - 新增 AND 信号量接口 `sem_p_multi`，支持原子申请多个信号量。
    - 新增 `semandtest` 并接入 `usertests`。
- **12/31:**
  - **内存管理：** 引入页面换入/换出机制。
    - 新增 swap 盘区与 bitmap 管理，预留连续 swap 区域。
    - 缺页异常路径支持 swapin，兼容懒分配与 COW。
    - 在 `uvmcopy` / `uvmunmap` 中处理 swapped 页的复制与释放。
    - 新增高强度 `swaptest` 并纳入 `usertests`。
  - **信号量集功能开发：**
    - `semset_create/semset_free/semset_p/semset_v/semset_p_multi`
    - 新增 `semsettest` 并接入 `usertests`。
  - **演示程序：**
    - 新增 `pcand` 生产者-消费者演示，展示“错误顺序导致死锁”和“AND 原子获取成功”。
  - **进程直接通信：**
    - 新增 `dmsgsend` / `dmsgrcv`，支持基于 PID 的直接消息通信。
    - 维护每进程消息缓冲队列，队列满/空时阻塞。
    - 新增 `dmsgtest` 并接入 `usertests`。
    - 修复回收时锁顺序，避免 `panic: acquire`。
    - 补充“接收超时发送阻塞”测试用例。
    - 修复 `freebytes` 统计空闲内存的锁顺序，避免并发野指针。
  - **文件系统：** 调整 buffer cache 回收流程（引入 `evict_lock`，避免跨桶双锁），补齐 mkfs 二级间接块写入路径。
- **12/31:**
  - **内存管理：** 实现 ELF 动态装入/按需装入。
    - `exec` 为可加载段建立 VMA，避免一次性分配全部物理页。
    - 缺页时从文件加载到内存映射。
    - 新增 `demandloadtest` / `bigbss` 并接入 `usertests`，覆盖 BSS 扩展场景。
