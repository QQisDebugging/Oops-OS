# 操作系统接口

### 1.操作系统接口

操作系统的工作：

- 让多个程序可以共享资源，为程序提供比硬件更好用的服务
- 抽象底层硬件，为软件隐藏硬件细节
- 多路复用：多个程序看起来可以同时运行
- 提供交互方式，程序可以共享数据，共同工作

> 操作系统通过==接口==为用户程序提供服务

内核：向运行中的程序提供服务的特殊程序

> 运行中的程序（进程），都拥有以下的内存空间：
>
> - 指令：程序运算
> - 数据：运算的变量
> - 栈：程序的过程调用

系统调用：进程通过系统调用使用内核服务。

> 系统调用会进入内核，让内核执行服务然后返回

硬件权限：内核有实现保护机制的硬件权限，而用户程序没有。因此内核使用了cpu的硬件保护机制来保证用户进程只能访问自己的内存空间

​	以下是内核提供的部分系统调用

| 系统调用                  | 描述                               |
| ------------------------- | ---------------------------------- |
| fork()                    | 创建进程                           |
| exit()                    | 结束当前进程                       |

### 2.进程和内存

xv6进程怎么组成的？

> - 用户内存空间（指令，数据，栈）
> - 进程状态（仅对内核可见）

xv6分时特性是什么

> 每个进程都有一个pid
>
> 分时复用的基本原理是将CPU的时间划分为多个时间片，每个时间片分配给不同的进程。
>
> 当一个进程不再执行的时候，xv6保存他的cpu寄存器；再次执行的时候恢复这些寄存器的值

系统调用fork

> fork：创建一个新的进程（子进程）
>
> - 两个返回：父进程中返回子进程的pid；子进程返回0
> - 子进程的内存==内容==和父进程一样（空间和寄存器不一样，改变一个不会影响另外一个）

> exit：停止调用它的进程，释放内存和打开的文件资源

> wait：返回当前进程已退出的子进程
>
> - 如果没有子进程退出，wait会一直等待

```c
int pid;
pid = fork();
if(pid > 0){
    printf("parent: child=%d\n", pid);
    pid = wait();
    printf("child %d is done\n", pid);
} else if(pid == 0){
    printf("child: exiting\n");
    exit();
} else {
    printf("fork error\n");
}
```

> exec：从文件读取内存镜像，替换调用者进程的内存空间
>
> - 以新的进程去代替原来的进程，但进程的PID保持不变
> - 参数1：可执行文件名
> - 参数2：字符串参数数组

```c
// 调用者程序会被替换为echo程序
char *argv[3];
argv[0] = "echo";
argv[1] = "hello";
argv[2] = 0;
exec("/bin/echo", argv);
printf("exec error\n");
```

### 2. IO和文件描述符

什么是文件描述符fd？

> 含义：是一个文件索引，用于指代可以被读写的文件
>
> - 每个进程都有一张表，文件描述符作为这张表的索引
>
> - 进程从文件描述符0读入（标准输入），从文件描述符1输出（标准输出），从文件描述符2输出错误（标准错误输出）

系统调用

> read(fd,buf,n)：从fd读最多n个字节，拷贝到buf，返回读出的字节
>
> - 文件描述符和偏移位置是对应的，当没有数据可读，就会返回0

> write(fd,buf,n)：从buf取n个字节到fd，并且返回实际写出的字节数
>
> - 在写的过程中，偏移会增加

> close（文件描述符）：释放文件描述符
>
> - 下一次分配的fd是当前进程最小的未被使用的fd

```c
// cat < input.txt
char *argv[2];
argv[0] = "cat";
### argv[1] = 0;
if(fork() == 0) {
    close(0); //子进程释放文件描述符0
    open("input.txt", O_RDONLY);// 新分配的文件描述符是最小未分配的fd=0
    exec("cat", argv);// 标准输入指向input.txt文件
}
```

> dup（文件描述符）：复制一个已有的文件描述符，返回一个指向同一个对象的新描述符
>
> - 这两个描述符共享文件偏移
> - 就是一个对象拥有了两个描述符

### 3. 管道

管道是什么？

> 是一个内核缓冲区，提供给进程一对文件描述符
>
> - 分别用于读和写操作

```c
// wc
int p[2];// 文件描述符对：p[0]读端口，p[1]写
char *argv[2];
argv[0] = "wc";
argv[1] = 0;
pipe(p);
if(fork() == 0) {
    // 让文件描述符0指向p[0]指向的对象（管道）
    close(0);
    dup(p[0]);
    close(p[0]);
    close(p[1]);
    // 从0中读取
    exec("/bin/wc", argv);
} else {
    write(p[1], "hello world\n", 12);
    close(p[0]);
    close(p[1]);
}
```

pipe和临时文件的区别？

> ```shell
> echo hello world | wc
> ```
>
> ```shell
> echo hello world > /tmp/xyz; wc < /tmp/xyz
> ```
>
> 管道会自动删除临时文件
>
> 管道可以传输任意长度的数据
>
> 管道允许同步（读操作会阻塞，直到write）

### 4. 文件系统

**文件**：文件就是简单的字节数组，主要用来存储数据。

**目录**：目录是包含指向文件和其他目录的引用的特殊文件。

**目录结构**：目录在文件系统中形成一棵树。

> chdir()：改变当前目录

> mkdir("/dir")：创建目录

> fd = open("/dir/file",O_CREATE|O_WRONGLY)：打开新的文件

> mknod("/console",1,1)：创建设备文件，参数为主设备号和辅设备号，唯一确定内核设备，内核将读写的系统调用转发到内核设备的实现上

> fstat:获取一个文件描述符指向的文件的信息
>
> 结构如下
>
> ```C
> #define T_DIR  1
> #define T_FILE 2
> #define T_DEV  3
> // Directory
> // File
> // Device
>      struct stat {
>        short type;  // 文件类型（目录、普通文件或设备）
>        int dev;     // 文件所在的磁盘设备
>        uint ino;    // 文件的 inode（文件在磁盘上的唯一标识）
>        short nlink; // 指向同一个 inode 的链接数。
>        uint size;   // 文件的字节大小。
> };
> ```

inode节点：唯一标识一个文件

> link("a","b")：创建b，指向a，使他们inode号相同
>
> unlink("a")：移除一个文件名（链接）
