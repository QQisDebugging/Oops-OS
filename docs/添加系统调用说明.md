## 系统调用

涉及到下面的文件

```c
syscall.c
syscall.h
usys.S
user.h
sysproc.c
proc.c
proc.h
defs.h
```

### 用户区修改

(1) 在user/user.h中添加系统调用函数的定义。

```c
int trace(int);
```

> 使得用户函数可以调用

(2) 在user/usys.pl中添加入口entry("trace")。

```c
entry("trace");
```

> 在make时会调用usys.pl生成user/usys.S汇编程序。在该汇编程序中，每个函数有五行，其中有三条指令，其功能是将系统调用号通过li(load imm)存入a7寄存器，之后使用ecall指令进入内核态，最后返回。

### 内核区修改

(1)在kernel/syscall.h中定义系统调用号。

```c
#define SYS_trace 23
```

(2)在kernel/syscall.c的syscalls函数指针数组中添加对应的函数。

```c
extern uint64 sys_trace(void);
```

```c
static uint64 (*syscalls[])(void) = {
    [SYS_fork] sys_fork,
    [SYS_exit] sys_exit,
    ......
    [SYS_trace] sys_trace,
}; // 这些索引会从1开始，不是从0开始
```

(3)在`kernel/sysproc.c`中添加该调用的实现：（也可以调用其他的内核函数，但是前提是放入def.h）

```c
uint64 sys_trace(void)
{ // 为当前进程的trace_mask赋值
  int n;
  if (argint(0, &n) < 0)
  { // n赋值为p->trapframe->a0，a0来自于进程用户空间，用与传参
    return -1;
  }
  myproc()->trace_mask = n; // trace_mask保存了a0的信息，用于调试
  return 0;
}
```
