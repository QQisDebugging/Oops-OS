# **define SYS_cps 22**

- **功能：遍历所有进程控制块，并打印相关信息**
- **输出：成功执行，返回22，系统调用号**

**实现思路：**

1. **定义结构体指针**
    - **定义了一个指向struct proc类型结构体的指针p，并初始化为指向名为proc的结构体**
2. **输出表头信息，并循环遍历**
    - **通过for循环从proc开始（即数组首地址），一直遍历到proc[NPROC]之前（因为数组下标是从 0 到NPROC - 1），每次循环指针p会指向下一个进程控制块结构体元素**
3. **输出进程状态信息**
    - **根据三种进程状态（SLEEPING、RUNNING、RUNNABLE）输出相应信息**
4. **如果成功返回22（系统调用号）**

**代码实现：**

```c
int cps(void)
{
  struct proc *p = proc; // 定义一个结构体(进程控制块)
  // stati();	// 中断
  printf("name \t pid \t state \t \t priority \tdyn_priority\n"); // 罗列所有的pid
  for (p = proc; p < &proc[NPROC]; p++)                           // NPROC为64
  {
    if (p->state == SLEEPING) // 睡眠
      printf("%s \t %d \t SLEEPING \t %d\t \t%d\n", p->name, p->pid, p->priority, p->dyn_priority);
    else if (p->state == RUNNING) // 正在执行
      printf("%s \t %d \t RUNNING \t %d\t \t%d\n", p->name, p->pid, p->priority, p->dyn_priority);
    else if (p->state == RUNNABLE) // 可运行队列
      printf("%s \t %d \t RUNNABLE \t %d\t \t%d\n", p->name, p->pid, p->priority, p->dyn_priority);
  }
  return 22; // 返回22
}
```

# **define SYS_trace 23**

- **功能：为当前进程创建一个一个用于调试的trace_mask值**
- **输入：用户传入的一个整数值**
- **输出：成功执行返回0**

**实现思路：**

1. **变量定义与参数获取**
  - **定义整形变量n，并通过argint将n赋值为p->trapframe->a0，a0来自于进程用户空间，用与传参**
2. **设置当前进程的trace_mask值**
  - **使用myproc()函数将n赋值给当前进程结构体中的trace_mask**
3. **执行成功返回0**

**代码实现：**

```c
uint64 sys_trace(void)
{ // 为当前进程的trace_mask赋值
  int n;
  if (argint(0, &n) < 0)
  { // n赋值为p->trapframe->a0，a0来自于进程用户空间，用与传参
    return -1;
  }
  myproc()->trace_mask = n; // trace_mask保存了a0的信息，用于调试
  return 0;
}
```

# **define SYS_sysinfo 24**

- **功能：查看当前进程数量已经剩余空间内存**
- **输入：系统资源信息**
- **输出：执行成功返回0**

**实现思路：**

1. **定义存储系统信息结构体并填充信息**
  - **定义了sysinfo info；**
  - **调用freebytes函数将获取到的内存信息填充**
  - **调用procnum函数，填充进程数量**
2. **获取目标用户的空间地址**
  - **调用argaddr(0, &dstaddr);将传入的用户空间地址的第0个地址传入dstaddr**
3. **将系统信息复制到用户空间**
    **-使用 copyout 函数尝试将内核空间中 info 结构体所存储的系统信息（包含了前面获取到的空闲内存、进程数量等内容）复制到用户空间中由 dstaddr 指定的地址处。**

**代码实现：**

```c
uint64 sys_sysinfo(void)
{
  struct sysinfo info;
  freebytes(&info.freemem);
  procnum(&info.nproc);

  // 获取虚拟地址
  uint64 dstaddr;
  argaddr(0, &dstaddr);

  // 从内核空间拷贝数据到用户空间
  if (copyout(myproc()->pagetable, dstaddr, (char *)&info, sizeof info) < 0)
    return -1;

  return 0;
}
```

# **define SYS_sbrk 12**

- **功能：使用懒分配，完成对进程内存空间大小的调整**
- **输入：进程内存的调整量**
- **输出：执行成功返回起始地址，失败返回-1**

**实现思路：**

1. **获取参数及错误处理**
  - **调用argint获取用户空间传入的第0个参数，并赋值给n**
2. **获取当前进程相关信息**
  - **调用myproc函数获取当前正在运行的进程的结构体指针p进而取得进程的相关信息**
3. **对内存空间的增减操作**
  - **根据传入的n的正负值对进程内存进行懒分配，执行相应增减操作**
4. **非法处理**
  - **如果在减少内存空间时，计算得到的新内存大小不大于 0（即 sz + n <= 0），这种情况被视为非法的内存调整操作，函数直接返回 -1，表示系统调用执行失败**
5. **返回调整前的内存起始地址**
  - **最后，无论内存是增加还是减少操作，只要前面的操作没有出现错误（返回 -1 的情况），函数都会返回 addr，也就是记录的调整前的内存起始地址，这个返回值可以让用户空间程序知晓之前内存的起始位置情况**

**代码实现：**

```c
uint64
sys_sbrk(void)
{
  int addr;
  int n;

  if (argint(0, &n) < 0)
    return -1;

  struct proc *p = myproc();
  addr = p->sz;
  uint64 sz = p->sz;

  if (n > 0)
  {
    // 懒分配
    p->sz += n;
  }
  else if (sz + n > 0)
  {
    sz = uvmdealloc(p->pagetable, sz, sz + n);
    p->sz = sz;
  }
  else
  {
    return -1;
  }
  return addr;
}
```

# **define SYS_setPriority 25**

- **功能：接收用户传入的进程标识符和优先级数值参数，对进程优先级进行设置**
- **输入：用户空间传入的整形参数**
- **输出：执行成功返回0，失败返回-1**

**实现思路：**

1. **获取参数**
  - **调用argint获取进程控制符pid、priority设定的优先级**
  - **如果出现返回值小于0的情况表示输入错误，直接返回-1**
2. **设定优先级**
  - **如果获取参数成功，则调用setPriority函数将pid与priority传入，并将priority的值，赋给进程控制符为pid的进程**
3. **执行成功返回0，失败返回-1**

**代码实现：**

```c
uint64 sys_setPriority(void)
{
  int pid, priority;

  // 从用户栈中读取 pid 和 priority 参数
  if (argint(0, &pid) < 0 || argint(1, &priority) < 0)
  {
    return -1; // 参数读取失败，返回错误
  }

  // 调用 setPriority 函数设置进程优先级
  return setPriority(pid, priority);
}
```

# **define SYS_execve 26**

- **功能：执行指定路径的程序**
- **输入：获取外部输入的地址与命令**
- **输出：执行成功调用exec函数执行指定程序，执行失败返回-1**

**实现思路：**

1. **参数获取与初始化**
  - **定义用于存储程序路径的字符数组 path 和指向参数字符串的指针数组 argv**
  - **通过 argstr 和 argaddr 函数按顺序获取程序路径和用户空间参数列表的起始地址，只要其中一个获取操作失败，就直接返回 -1，不再继续后续流程**
  - **取成功后，将程序路径 path 赋值给 argv[0]，作为参数列表的第一个元素，也就是将要执行程序的路径本身**
2. **构建参数列表**
  - **进入循环来处理参数列表中的其他参数（从索引 1 开始），循环条件为持续进行直到遇到结束条件break退出**
  - **在循环中，先进行边界检查，如果参数索引 i 超出了 argv 数组所能容纳的最大元素个数（通过 NELEM(argv) 判断），则跳转到错误处理部分（bad）**
  - **接着使用 fetchaddr 函数获取每个参数在用户空间的实际地址（通过计算偏移量 uargv + sizeof(uint64) * (i - 1) 来定位每个参数的地址**
  - **当获取到的参数地址对应的 uarg 值为 0 时，意味着参数列表结束，将当前 argv 元素赋值为 0 来标记参数列表的结尾，然后跳出循环**
  - **对于每个非空的参数，先调用 kalloc 函数为其分配内核内存空间（用于存放从用户空间获取的参数字符串内容），若内存分配失败则跳转到错误处理部分。分配成功后，使用 fetchstr 函数从用户空间对应的地址 uarg 处获取字符串内容，并存储到刚分配好的内存中（通过 argv[i] 指向的内存区域）**
  3. **程序执行与内存清理**

- **完成参数列表构建后，调用 exec(path, argv) 函数来执行指定的程序，传递构建好的程序路径和参数列表进去**
- **在 exec 函数执行完毕后（无论执行成功与否），需要清理之前为参数分配的内存（除了 argv[0] 对应的程序路径，因为它不需要清理，从索引 1 开始清理），通过循环逐个调用 kfree 函数释放内存。最后返回 exec 函数的返回值作为系统调用的最终结果返回给调用者**

  4. **错误处理**

- **代码中有一个标记为 bad 的错误处理部分，当在参数处理过程中出现各种错误（如内存分配失败、获取字符串内容失败等）时会跳转到此处。在这里同样通过循环逐个调用 kfree 函数清理已经为参数分配的内存（只要参数不为空），然后返回 -1，表示系统调用执行失败，以此来保证在出现异常情况时能合理地释放资源并向调用者反馈错误信息**

**代码实现：**

```c
uint64
sys_execve(void)
{
  char path[260], *argv[MAXARG];
  int i;
  uint64 uargv, uarg;
  // 获取路径和参数地址
  if (argstr(0, path, 260) < 0 || argaddr(1, &uargv) < 0)
  {
    return -1;
  }
  memset(argv, 0, sizeof(argv));
  // 设置 argv[0] 为程序的路径
  argv[0] = path;
  // 从用户空间获取其他参数
  for (i = 1;; i++)
  {
    if (i >= NELEM(argv))
      goto bad;
    // 获取每个参数的地址
    if (fetchaddr(uargv + sizeof(uint64) * (i - 1), (uint64 *)&uarg) < 0)
      goto bad;
    // 如果参数为空，结束循环
    if (uarg == 0)
    {
      argv[i] = 0;
      break;
    }
    // 为参数分配内存
    argv[i] = kalloc();
    if (argv[i] == 0)
      goto bad;
    // 获取字符串内容
    if (fetchstr(uarg, argv[i], PGSIZE) < 0)
      goto bad;
  }
  // 打印参数（调试用）
  // for (i = 0; i < NELEM(argv) && argv[i] != 0; i++)
  // {
  //   printf("argv[%d]: %s\n", i, argv[i]);
  // }
  // 调用 exec 执行程序
  // printf("%s", path);
  int ret = exec(path, argv);
  // 清理已分配的内存
  for (i = 1; i < NELEM(argv) && argv[i] != 0; i++)
    kfree(argv[i]);
  return ret;
bad:
  // 清理已分配的内存
  for (i = 0; i < NELEM(argv) && argv[i] != 0; i++)
    kfree(argv[i]);
  return -1;
}
```

# **define SYS_print_pgtable 28**

- **功能：打印当前进程的页表信息**
- **输入：当前进程的结构体指针**
- **输出：执行成功返回0**

**实现思路：**

1. **获取当前进程结构体指针**
  - **调用 myproc 函数获取当前正在运行的进程的结构体指针，并赋值给变量p**
2. **获取锁确保互斥访问**
  - **获取当前进程结构体中的锁（p->lock），确保可以处于可以访问的状态**
3. **打印页表信息**
  - **调用 vmprint 函数，并传入当前进程的页表指针（p->pagetable），由 vmprint 函数负责实际的页表信息打印工作**
4. **释放锁、**
  - **在完成打印页表信息后，立即释放当前进程的锁以供其他相关进程可以再次获取锁**
**5.返回执行结果**
  - **如果成功执行返回0**

**代码实现：**

```c
uint64 sys_print_pgtable(void)
{
  struct proc *p = myproc();
  acquire(&p->lock);
  vmprint(p->pagetable);
  release(&p->lock);
  return 0;
}
```

# **define SYS_mmap 29**

- **功能：建立文件映射**
- **输入：内存映射的具体要求与配置情况**
- **输出：执行成功返回映射区域在进程虚拟地址空间中的起始虚拟地址，执行失败返回预先定义错误err**

**实现思路：**

1. **参数定义与获取**
  - **首先定义了一系列变量用于存储系统调用传入的参数，包括 addr（期望的映射起始虚拟地址）、length（映射区域的长度）等相关信息**
  - **通过多个函数（argaddr、argint、argfd 等）从外部获取系统调用的参数，并进行错误判断**
2. **参数合法性检查**
  - **对 addr、offset 和 length 参数做进一步的合法性判断**
3. **权限与文件可写性检查**
  - **检查传入的参数中关于内存保护权限（prot）、映射标志（flags）以及文件可写性（vfile->writable）之间的兼容性**
4. **虚拟内存空间检查**
  - **获取当前进程的结构体指针 p = myproc()，然后检查当前进程的虚拟内存空间是否足够容纳要映射的区域**
5. **虚拟内存区域分配与设置**
  - **通过循环遍历查找当前进程中未使用的虚拟内存区域结构体**
  - **当找到一个未使用的 VMA 结构体（p->vma[i].used == 0）时,标记改VMA结构体已使用，并对映射区域进行配置，再通过filedup增加关联文件引用技术，最后更新当前进程已使用内存大小，返回该VMA结构体的其实虚拟机地址**
6. **映射失败情况处理**
  - **如果遍历完所有的 VMA 结构体都没有找到未使用的，说明无法为此次映射分配合适的虚拟内存区域，那么函数就返回表示错误的 err 值**

**代码实现：**

```c
uint64
sys_mmap(void)
{
  uint64 addr;
  int length;
  int prot;
  int flags;
  int vfd;
  struct file *vfile;
  int offset;
  uint64 err = 0xffffffffffffffff;

  // 获取系统调用参数
  if (argaddr(0, &addr) < 0 || argint(1, &length) < 0 || argint(2, &prot) < 0 ||
      argint(3, &flags) < 0 || argfd(4, &vfd, &vfile) < 0 || argint(5, &offset) < 0)
    return err;

  // 实验提示中假定addr和offset为0，简化程序可能发生的情况
  if (addr != 0 || offset != 0 || length < 0)
    return err;

  // 文件不可写则不允许拥有PROT_WRITE权限时映射为MAP_SHARED
  if (vfile->writable == 0 && (prot & PROT_WRITE) != 0 && flags == MAP_SHARED)
    return err;

  struct proc *p = myproc();
  // 没有足够的虚拟地址空间
  if (p->sz + length > MAXVA)
    return err;

  // 遍历查找未使用的VMA结构体
  for (int i = 0; i < NVMA; ++i)
  {
    if (p->vma[i].used == 0)
    {
      p->vma[i].used = 1;
      p->vma[i].addr = p->sz;
      p->vma[i].len = length;
      p->vma[i].flags = flags;
      p->vma[i].prot = prot;
      p->vma[i].vfile = vfile;
      p->vma[i].vfd = vfd;
      p->vma[i].offset = offset;

      // 增加文件的引用计数
      filedup(vfile);

      p->sz += length;
      return p->vma[i].addr;
    }
  }

  return err;
}
```

# **define SYS_munmap 30**

- **功能：用于取消文件映射**
- **输入：需要取消映射的其实虚拟地址以及需要取消映射的长度**
- **输出：执行成功返回0，异常返回-1**

**实现思路：**

1. **参数获取**
  - **定义两个变量addr、length分别表示要取消映射的起始虚拟地址和要取消映射区域长度。并通过argaddr、argint两个函数从外部获取，并对获取失败进行-1返回**
2. **遍历虚拟内存区域（VMA）查找匹配项**
  - **获取当前进程的结构体指针 p = myproc()**
  - **循环遍历当前进程中所有的虚拟内存区域结构体（VMA）**
  - **对每个VMA进行条件判断，p->vma[i].used为真，p->vma[i].len >= length**
3. **根据地址范围调整 VMA 信息**
  - **从起始地址开始**
  - **从结束位置开始**
4. **处理未找到匹配 VMA 的情况**
  - **若遍历完成i == NVMA表示没有匹配项，返回-1**
5. **将共享可写页面写回文件系统**
  - **如果找到的 VMA 满足是共享映射（p->vma[i].flags == MAP_SHARED）并且具有可写权限（(p->vma[i].prot & PROT_WRITE)!= 0），则调用 filewrite 函数将对应页面的内容写回关联的文件系统中**
6. **解除页面映射**
  - **调用 uvmunmap 函数来解除从起始地址 addr 开始，长度为 length / PGSIZE 个页面**
7. **清理 VMA 相关资源**
8. **返回操作结果**

**代码实现：**

```c
uint64
sys_munmap(void)
{
  uint64 addr;
  int length;
  if (argaddr(0, &addr) < 0 || argint(1, &length) < 0)
    return -1;

  int i;
  struct proc *p = myproc();
  for (i = 0; i < NVMA; ++i)
  {
    if (p->vma[i].used && p->vma[i].len >= length)
    {
      // 根据提示，munmap的地址范围只能是
      // 1. 起始位置
      if (p->vma[i].addr == addr)
      {
        p->vma[i].addr += length;
        p->vma[i].len -= length;
        break;
      }
      // 2. 结束位置
      if (addr + length == p->vma[i].addr + p->vma[i].len)
      {
        p->vma[i].len -= length;
        break;
      }
    }
  }
  if (i == NVMA)
    return -1;

  // 将MAP_SHARED页面写回文件系统
  if (p->vma[i].flags == MAP_SHARED && (p->vma[i].prot & PROT_WRITE) != 0)
  {
    filewrite(p->vma[i].vfile, addr, length);
  }

  // 判断此页面是否存在映射
  uvmunmap(p->pagetable, addr, length / PGSIZE, 1);

  // 当前VMA中全部映射都被取消
  if (p->vma[i].len == 0)
  {
    fileclose(p->vma[i].vfile);
    p->vma[i].used = 0;
  }

  return 0;
}
```

# **define SYS_sem_p 35**

- **功能：执行信号量的P操作**
- **输入：整型参数id，需要操作的信号量在信号量数组中索引**
- **输出：成功执行返回0， 失败返回-1**

**实现思路：**

1. **获取参数并进行合法性检查**
- **成功获取id，并检查id>0,且<SEM_MAX_NUM**
- **使用信号量锁作为 sleep 的互斥锁，防止丢失唤醒**
2. **获取相关锁**
  - **获取进程锁的目的是为了保证在对信号量进行操作以及后续可能涉及到的进程状态变更等操作时，不会被其他执行单元干扰，确保操作的原子性和互斥性。**
3. **获取信号量资源并处理**
  - **释放锁，避免对其他执行单元阻塞**
  - **调用sleep让当前进程进入休眠状态**
  - **如果 sems[id].resource_count >= 0，即资源足够，说明当前进程成功获取到了信号量资源，调用release(sems[id].lock)释放信号量锁**
4. **释放进程锁并返回结果**

**代码实现:**

```c
int sys_sem_p()
{
  int id;
  struct proc *p = myproc();

  if (argint(0, &id) < 0 || id < 0 || id >= SEM_MAX_NUM) // 参数合法性检查
  {
    return -1;
  }

  // printf("sem_p: 尝试获取信号量 id = %d\n", id);

  acquire(&p->lock);       // 获取当前进程的锁
  acquire(&sems[id].lock); // 获取信号量锁

  sems[id].resource_count--;
  if (sems[id].resource_count < 0)
  {
    release(&sems[id].lock);    // 释放信号量锁
    sleep(&sems[id], &p->lock); // 使用进程锁进行休眠
  }
  else
  {
    release(&sems[id].lock); // 如果资源足够，释放信号量锁
  }

  release(&p->lock); // 释放进程锁
  return 0;
}
```

4. **执行成功返回文件描述符fd**

```c
int sys_connect(void)

}
```

# **define SYS_mqget 46**

- **功能：获取或创建一个基于给定 key 的消息队列**
- **输入：key(消息队列标识符)**
- **输出：执行成功返回消息队列在 mqs[] 数组中的下标（idx），失败返回-1**

**实现思路：**

1. **获取锁**
  - **调用mqloc锁来保证检查和修改队列的互斥访问**
2. **查找消息队列**
  - **调用findkey(key)，查找是否已存在相应消息队列**
  - **如果没找到则创建一个新队列**
3. **判断进程是否已经使用该消息队列**
  - **如果当前进程未使用该消息队列，则通过 proc->mqmask |= 1 << idx 将该进程标记为使用这个消息队列**
  - **增加引用计数**
4. **消息队列未找到**
  - **调用 newmq(key) 创建一个新的消息队列**
5. **释放锁，并返回**

**代码实现：**

```c
int sys_mqget(uint key)
{
    acquire(&mqlock);
    int idx = findkey(key);
    if (idx != -1)
    { // 如果key对应的消息队列已经创建
        if (!(proc->mqmask >> idx & 1))
        {
            proc->mqmask |= 1 << idx; // 标记该进程使用该消息队列
            mqs[idx].refcount++;      // 消息队列的引用计数+1
        }
        release(&mqlock);
        return idx;
    }

    // 对应key消息队列未创建则newmg创建
    idx = newmq(key); // 创建消息队列
    release(&mqlock);
    return idx; // 返回该消息队列在mqs[]中的下标
}
```

# **define SYS_msgsnd 47**

- **功能：消息队列的管理（包括消息存储、进程阻塞与唤醒等）**
- **输入：mqid(消息队列id)、msg(消息数据指针)、sz(消息字节数)**
- **输出：消息发送成功返回0，失败返回-1**

**实现思路：**

1. **建业消息合法性**
  - **检查消息队列id是否有效**
  - **检查消息队列是否存在**
2. **解析传入的消息数据**
  - **消息数据 msg 的前 4 字节用来存储消息类型，因此通过 msg + 4 获取实际消息内容**
  - **将消息的前 4 字节（即 msg 指向的第一个整数）作为消息类型，保存在 type 中**
3. **检查消息队列是否为空**
4. **获取锁并进行消息发送**
  - **若消息队列有足够空间，则在队尾空闲消息区插入新消息并更新状态**
5. **队列空间不足则阻塞进程**

**代码实现：**

```c
int sys_msgsnd(uint mqid, void *msg, int sz)
{
    // 校验消息队列的合法性
    if (mqid < 0 || mqid >= MQMAX || mqs[mqid].status == 0)
    {
        return -1;
    }

    // 解析传入的消息数据
    char *data = (char *)(((int *)(msg + 4)));

    int *type = ((int *)msg); // 获取消息类型

    // 如果队列为空，打印错误信息
    if (mqs[mqid].msgs == 0)
    {
        printf("msgsnd failed: msgs == 0.\n");
        return -1;
    }

    acquire(&mqlock);

    while (1)
    { // 一直循环直到发送成功
        if (mqs[mqid].curbytes + sz + 16 <= mqs[mqid].maxbytes)
        { // 如果剩余空间充裕
            struct msg *m = mqs[mqid].msgs;

            // 找到队尾最后一个空闲消息区
            while (m->next != 0)
            {
                m = m->next;
            }
            // 退出循环时，m->next == 0，表示空闲消息区

            m->next = (void *)m + m->datasize + 16; // 计算用于存储消息的起始位置
            m = m->next;                            // m为本消息存储空间起点

            m->type = *(type);            // 填写本消息的类型
            m->next = 0;                  // 本消息暂无后续消息
            m->dataaddr = (void *)m + 16; // 数据区的起始位置
            m->datasize = sz;             // 数据长度

            // 复制消息数据到消息区
            memmove(m->dataaddr, data, sz);

            mqs[mqid].curbytes += (sz + 16); // 可用空间缩减

            // 唤醒所有读阻塞进程
            for (int i = 0; i < rstart; i++)
            {
                wakeup(rqueue[i]);
            }

            rstart = 0; // 读阻塞队列置空
            release(&mqlock);
            return 0; // 成功发送消息
        }
        else
        { // 如果空间不足，进程睡眠在wqueue阻塞队列
            printf("msgsnd: cannot alloc: pthread: %d sleep.\n", myproc()->pid);
            wqueue[wstart++] = proc; // 将当前进程加入等待队列

            sleep(proc, &mqlock); // 进入休眠状态，等待唤醒
        }
    }
    return -1; // 发送失败
}
```

# **define SYS_msgrcv 48**

- **功能：从指定的消息队列中接收消息**
- **输入：mqid(消息队列id)、msq(消息缓冲区指针)、sz(目标缓冲区大小)**
- **输出：若成功读取消息返回0，若执行发生错误或消息队列没有匹配消息，返回-1**

**实现思路：**

1. **检验消息队列合法性**
  - **检查消息队列id是否合法**
  - **校检消息队列状态是否有效**
2. **解析消息数据**
  - **消息类型被存储在 msg 的前 4 字节中，因此将 msg 的地址转换为 int* 类型，读取消息类型**
  - **数据部分从 msg + 4 开始，作为消息内容存储**
3. **获取锁并读取消息**
  - **遍历消息队列，查找匹配的消息类型**
  - **找到符合条件的消息，先将内容复制到目标缓冲区**
  - **将消息从队列中移除**
  - **更新当前队列字节数**
  - **调用reloc对消息队列进行内存整理**
  - **唤醒所用因队列满而处于阻塞状态的写进程**
  - **释放锁并返回0**
4. **队列中没用符合的消息时，进入 阻塞**
  - **返回-1**

**代码实现：**

```c
int sys_msgrcv(uint mqid, void *msg, int sz)
{
    // 校验消息队列的合法性
    if (mqid < 0 || mqid >= MQMAX || mqs[mqid].status == 0)
    {
        return -1;
    }

    int *type = msg;     // 待读取消息类型
    int *data = msg + 4; // 待读取消息目标位置

    acquire(&mqlock);

    while (1)
    {
        struct msg *m = mqs[mqid].msgs->next;
        struct msg *pre = mqs[mqid].msgs;

        while (m != 0)
        {
            if (m->type == *type)
            { // 找到要读取的消息类型
                // 复制消息数据到目标位置
                memmove(data, m->dataaddr, sz);

                // 从队列中删除已读取的消息
                pre->next = m->next;

                // 释放消息占用的空间
                mqs[mqid].curbytes -= (m->datasize + 16);

                // 重新整理内存
                reloc(mqid);

                // 唤醒所有写阻塞进程
                for (int i = 0; i < wstart; i++)
                {
                    wakeup(wqueue[i]);
                }
                wstart = 0; // 写阻塞队列置空

                release(&mqlock);
                return 0; // 成功读取消息
            }

            pre = m;
            m = m->next;
        }

        // 如果没有找到匹配的消息类型，当前进程进入休眠状态
        printf("msgrcv: can not read: pthread: %d sleep.\n", proc->pid);

        // 将当前进程加入读阻塞队列
        rqueue[rstart++] = proc;

        sleep(proc, &mqlock); // 进入休眠状态，等待唤醒
        return -1;            // 未能成功读取消息
    }
}
```

# **define SYS_clone 52**

- **功能：创建一个具有自己的堆栈和程序计数器等信息当前进程的副本**
- **输入：fcn(要执行的函数指针)、arg(传递给函数fcn的参数)、stack(新进程的起始地址)**
- **输出：执行成功返回新线程标识符，失败返回-1**

**实现思路：**

1. **分配新进程结构体**
2. **继承父进程的资源，设置新进程的堆栈**
3. **复制父进程的上下文**
4. **设置程序计数器和堆栈指针**
5. **复制堆栈内容到用户空间**
6. **文件描述符和工作目录的继承**
7. **释放锁并返回进程 ID**

```c
int sys_clone(void)
{
  uint64 fcn;
  uint64 arg;
  uint64 stack;
  argaddr(0,&fcn);
  argaddr(1,&arg);
  argaddr(2,&stack);
  return clone(fcn,arg,stack);
}
int clone(uint64 fcn, uint64 arg, uint64 stack)
{
  struct proc *curproc = myproc();  // 获取当前进程的结构体指针
  struct proc *np = 0;              // 创建新的进程指针
  // 分配新的进程结构体，如果失败则返回-1
  if ((np = allocproc()) == 0) return -1;
  np->pagetable = curproc->pagetable;   // 子进程继承父进程的页表
  np->sz = curproc->sz;                 // 子进程继承父进程的地址空间大小
  np->pthread = curproc;                // 子进程关联到父进程，表示它是一个线程
  np->ustack = (void *)stack;           // 设置子进程的用户堆栈地址
  np->parent = 0;                       // 目前没有父进程（这里应该指向父进程，可能是逻辑问题）
  // 复制父进程的 trapframe（陷阱帧），包含上下文信息（如寄存器值）
  *np->trapframe = *curproc->trapframe;
  // 为新的进程分配内核栈空间
  void *stackin = kalloc();
  // 设置栈指针（从栈顶开始，倒推16字节位置）
  uint64 *sp = stackin + 4096 - 16;
  // 在内核栈中伪造现场，假装子进程的返回地址是 fcn 函数，并将堆栈指针指向线程栈
  np->trapframe->epc = fcn;             // 设置程序计数器为目标函数地址
  np->trapframe->sp = stack + 4096 - 16; // 设置栈指针为线程栈的起始位置
  np->trapframe->s0 = stack + 4096 - 16; // 设置帧指针为线程栈的起始位置
  np->trapframe->a0 = 0;                // 设置返回值寄存器为 0
  // 在内核栈中伪造函数参数
  *(sp + 1) = arg;  // 将参数 `arg` 存入栈中
  *sp = 0xffffffffffffffff;  // 设置一个伪造的返回地址，表示函数调用的结束
  // 将内核栈的内容复制到用户栈中
  copyout(curproc->pagetable, stack, stackin, PGSIZE);
  // 复制文件描述符：子进程继承父进程的打开文件
  for (int i = 0; i < NOFILE; i++) {
    if (curproc->ofile[i])
      np->ofile[i] = filedup(curproc->ofile[i]);
  }
  // 复制父进程的当前工作目录
  np->cwd = idup(curproc->cwd);
  // 复制父进程的进程名
  safestrcpy(np->name, curproc->name, sizeof(curproc->name));
  // 释放新进程的锁
  release(&np->lock);
  int pid = np->pid;  // 获取新进程的 PID
  // 重新获取新进程的锁并将状态设置为 RUNNABLE，表示它可以被调度
  acquire(&np->lock);
  np->state = RUNNABLE;
  // 释放进程锁
  release(&np->lock);
  return pid;  // 返回新进程的 PID
}
```

# **define SYS_join 53**

- **功能：等待当前进程的子线程结束并回收资源**
- **输入：stackaddr(用户空间的堆栈地址)**
- **输出：执行成功返回子线程的退出状态或相关信息**

**实现思路：**

1. **遍历进程列表，查找子进程**
2. **检查子进程状态，清理子进程资源**
3. **返回堆栈地址给父进程**
4. **释放子进程锁并释放当前进程锁**
5. **如果没有子进程，或者当前进程被杀死**
6. **等待子进程**
7.  **结束循环**

```c
int sys_join(void)
{
  uint64 stackaddr;
  argaddr(0,&stackaddr);
  return join(stackaddr);
}
int join(uint64 stackaddrout)
{
  uint64 stackaddrin;  // 用于存储子进程的堆栈地址
  struct proc *curproc = myproc();  // 获取当前进程
  struct proc *p;
  int havekids;
  acquire(&curproc->lock);  // 获取当前进程的锁，防止其状态被修改
  //循环等待子进程结束
  for(;;)
  {
    havekids = 0;  // 重置是否有子进程的标志
    for(p = proc; p < &proc[NPROC]; p++)  // 遍历所有进程
    {
      if(p->pthread == curproc)  // 判断该进程是否为当前进程的子进程（线程）
      {
        acquire(&p->lock);  // 获取子进程的锁
        havekids = 1;  // 当前进程有子进程
        if(p->state == ZOMBIE)  // 如果子进程已经结束
        {
          stackaddrin = (uint64)p->ustack;  // 获取子进程的堆栈地址
          int pid = p->pid;  // 获取子进程的PID
          // 释放子进程的内核栈
          kfree((void*)p->kstack);
          p->kstack = 0;
          // 重置子进程的状态和相关字段
          p->state = UNUSED;
          p->pid = 0;
          p->parent = 0;
          p->pthread = 0;
          p->name[0] = 0;
          p->killed = 0;
          // 将子进程的堆栈地址返回给父进程
          copyout(p->pagetable, stackaddrout, (char*)&stackaddrin, 8);
          release(&p->lock);  // 释放子进程的锁
          release(&curproc->lock);  // 释放当前进程的锁
          return pid;  // 返回子进程的PID，表示成功回收
        }
        release(&p->lock);  // 如果子进程不是ZOMBIE，释放子进程锁
      }
    }
    // 如果没有子进程或者当前进程已被杀死，返回失败
    if(!havekids || curproc->killed) {
      release(&curproc->lock);
      return -1;
    }
    // 如果有子进程但未结束，进入休眠
    sleep(curproc, &curproc->lock);
  }
  return 0;  // 这是一个永远不会到达的地方
}
```
