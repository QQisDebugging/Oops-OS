### 2.18 位置读写（pread/pwrite）

#### 2.18.1 设计背景

传统的 `read()` 和 `write()` 系统调用会修改文件描述符的当前偏移量，这在多线程环境下会导致竞态条件。如果多个线程共享同一个文件描述符并同时进行读写操作，它们会相互干扰文件偏移量，导致数据损坏或读取错误的数据。

`pread()` 和 `pwrite()` 是 POSIX 标准定义的原子位置读写接口，允许在指定偏移位置进行 I/O 操作而不改变文件偏移量，完美解决了多线程并发访问同一文件描述符的问题。

#### 2.18.2 接口与语义

**系统调用号：** 
- `SYS_pread = 92`
- `SYS_pwrite = 93`

```c
// 在指定偏移位置读取数据，不改变文件偏移量
// 返回实际读取的字节数，失败返回 -1
int pread(int fd, void *buf, int count, int offset);

// 在指定偏移位置写入数据，不改变文件偏移量
// 返回实际写入的字节数，失败返回 -1
int pwrite(int fd, const void *buf, int count, int offset);
```

**参数说明：**
- `fd`: 已打开的文件描述符
- `buf`: 读取/写入的数据缓冲区
- `count`: 要读取/写入的字节数
- `offset`: 从文件开头的绝对偏移量（必须 >= 0）

**语义保证：**
1. 偏移量独立：操作使用指定的 offset，不读取也不修改 fd 的当前偏移量
2. 原子性：整个操作是原子的，不会被其他线程的文件操作干扰
3. 可写入超出文件末尾：pwrite 可以扩展文件大小
4. EOF 处理：pread 读取超出文件末尾时返回实际可读取的字节数

#### 2.18.3 实现策略

**sys_pread 实现要点：**
```c
uint64 sys_pread(void) {
    struct file *f;
    uint64 buf;
    int count, offset;
    
    // 获取参数
    argfd(0, 0, &f);
    argaddr(1, &buf);
    argint(2, &count);
    argint(3, &offset);
    
    // 参数校验
    if (offset < 0 || count < 0) return -1;
    if (f->readable == 0) return -1;
    if (f->type != FD_INODE) return -1;
    
    // 直接调用 readi，不修改 f->off
    struct inode *ip = f->ip;
    ilock(ip);
    int r = readi(ip, 1, buf, offset, count);
    iunlock(ip);
    
    return r;
}
```

**sys_pwrite 实现要点：**
```c
uint64 sys_pwrite(void) {
    struct file *f;
    uint64 buf;
    int count, offset;
    
    // 获取参数并校验
    argfd(0, 0, &f);
    argaddr(1, &buf);
    argint(2, &count);
    argint(3, &offset);
    
    if (offset < 0 || count < 0) return -1;
    if (f->writable == 0) return -1;
    if (f->type != FD_INODE) return -1;
    
    // 分批写入，避免超过日志事务大小限制
    int max = ((MAXOPBLOCKS-4) * BSIZE);
    int total = 0;
    struct inode *ip = f->ip;
    
    while (total < count) {
        int n = count - total;
        if (n > max) n = max;
        
        begin_op();
        ilock(ip);
        int r = writei(ip, 1, buf + total, offset + total, n);
        iunlock(ip);
        end_op();
        
        if (r != n) break;
        total += r;
    }
    
    return total;
}
```

**关键设计决策：**
1. 不锁定文件偏移量，只锁定 inode 进行实际 I/O
2. pwrite 分批处理大写入，每批独立事务
3. 复用现有的 readi/writei 函数，最小化代码改动

#### 2.18.4 测试

`/user/test/preadwritetest` 覆盖：

- 基本 pwrite：验证写入位置正确，偏移量不变
- 基本 pread：验证读取位置正确，偏移量不变
- 模拟并发：多次 pwrite/pread 不同位置，验证独立性
- 边界情况：EOF 处理、负偏移拒绝、文件扩展
- 大偏移：在块边界位置读写

#### 2.18.5 使用场景

1. **数据库系统**：直接 I/O 访问数据文件的特定页面
2. **多线程日志**：多线程同时追加日志到不同位置
3. **文件服务器**：并发处理多个客户端对同一文件的随机访问请求
4. **内存映射替代**：在不支持 mmap 的环境下实现类似功能

#### 2.18.6 优点

- **线程安全**：无需额外同步即可多线程并发访问
- **高效**：减少 lseek + read/write 的系统调用开销
- **简单**：API 直观，易于使用
- **兼容**：符合 POSIX 标准，便于移植应用程序
