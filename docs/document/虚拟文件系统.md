# 虚拟文件系统 (VFS) 设计文档

## 1. 概述

虚拟文件系统 (Virtual File System, VFS) 是一个抽象层，允许不同的文件系统（如 xv6 原生文件系统、FAT 等）通过统一的接口被访问。VFS 使得操作系统可以支持多种文件系统格式，并允许用户在运行时挂载和卸载不同的文件系统。

## 2. 架构设计

### 2.2 数据结构

#### vfs_inode (通用 inode)
```c
struct vfs_inode {
  uint dev;           // 设备号
  uint inum;          // inode 号
  int ref;            // 引用计数
  int valid;          // 有效标志
  int type;           // 文件类型
  int size;           // 文件大小
  int nlink;          // 硬链接数
  void *fs_inode;     // 指向具体文件系统的 inode
  struct fs_operations *ops;  // 文件系统操作
};
```

#### vfs_mount (挂载点)
```c
struct vfs_mount {
  int active;                   // 是否激活
  char source[MAXPATH];         // 源设备
  char target[MAXPATH];         // 挂载路径
  int fs_type;                  // 文件系统类型
  struct fs_operations *ops;    // 文件系统操作
  void *fs_data;                // 文件系统私有数据
};
```

#### fs_operations (文件系统操作接口)
```c
struct fs_operations {
  int fs_type;
  char name[16];
  
  // inode 操作
  struct vfs_inode* (*iget)(struct vfs_mount *mnt, uint inum);
  void (*iput)(struct vfs_inode *ip);
  void (*ilock)(struct vfs_inode *ip);
  void (*iunlock)(struct vfs_inode *ip);
  
  // 数据操作
  int (*readi)(struct vfs_inode *ip, int user_dst, uint64 dst, uint off, uint n);
  int (*writei)(struct vfs_inode *ip, int user_src, uint64 src, uint off, uint n);
  
  // 目录操作
  struct vfs_inode* (*dirlookup)(struct vfs_inode *dp, char *name, uint *off);
  int (*dirlink)(struct vfs_inode *dp, char *name, uint inum);
  
  // 文件创建/删除
  struct vfs_inode* (*create)(struct vfs_mount *mnt, char *path, int type);
  int (*unlink)(struct vfs_mount *mnt, char *path);
  
  // 路径解析
  struct vfs_inode* (*namei)(struct vfs_mount *mnt, char *path);
  
  // 挂载/卸载
  int (*mount)(struct vfs_mount *mnt);
  int (*unmount)(struct vfs_mount *mnt);
};
```

## 3. 系统调用

### 3.1 mount
```c
int mount(const char *source, const char *target, const char *fstype);
```
- `source`: 源设备路径（如 "/dev/disk1"）
- `target`: 挂载目标路径（如 "/mnt"）
- `fstype`: 文件系统类型（"xv6" 或 "fat"）
- 返回值: 成功返回 0，失败返回 -1

### 3.2 umount
```c
int umount(const char *target);
```
- `target`: 挂载点路径
- 返回值: 成功返回 0，失败返回 -1

## 4. 支持的文件系统

### 4.1 xv6 原生文件系统 (FS_TYPE_XV6)
- 作为根文件系统自动挂载
- 完全兼容现有的 xv6 文件系统实现
- 通过适配器层对接 VFS 接口

### 4.2 FAT16 文件系统 (FS_TYPE_FAT)
- 支持 FAT16 格式的磁盘镜像
- 特性：
  - 512 字节扇区大小
  - 8.3 短文件名格式
  - 簇链管理
  - FAT 表缓存
  - 目录项解析
- 限制：
  - 当前不支持长文件名 (LFN)
  - 最大文件大小受 FAT16 限制

## 5. 路径解析

VFS 层使用最长前缀匹配算法来确定文件路径属于哪个挂载点：

```c
struct vfs_mount* vfs_get_mount(char *path)
```

例如：
- 路径 "/mnt/data/file.txt"，如果 "/mnt" 已挂载 FAT 文件系统
- 则该路径将由 FAT 文件系统处理

## 6. 使用示例

```c
// 挂载 FAT 文件系统
int ret = mount("/dev/disk1", "/mnt", "fat");
if (ret < 0) {
    printf("mount failed\n");
}

// 访问挂载的文件系统
int fd = open("/mnt/readme.txt", O_RDONLY);
// ...

// 卸载文件系统
ret = umount("/mnt");
```

## 8. 扩展指南

### 添加新的文件系统支持

1. 在 `kernel/include/` 创建头文件定义数据结构
2. 在 `kernel/filesystem/` 实现文件系统驱动
3. 实现 `fs_operations` 所有接口函数
4. 在初始化时调用 `vfs_register_fs()` 注册
5. 更新 Makefile 编译新文件
